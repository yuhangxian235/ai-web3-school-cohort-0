## 📝 三个关键概念

### ① Token
> Token 是模型处理文本的基本单位。代码和 JSON 的符号、括号、冒号等每个都会成为独立 token，所以比普通文本更"吃" token。100 个中文字 ≈ 150-200 token，但同样长度的 JSON 可能 300+ token。

### ② Hallucination（幻觉）
> 模型生成"看起来合理但不真实"的内容——编造 API、错误解释代码、引用不存在资料。**Prompt 无法完全避免幻觉**，因为 LLM 本质是概率预测，不是查数据库。解决方案是外部校验：链上数据、schema 验证、人工确认。

### ③ AI 分析交易时的行为边界
> Prompt 里必须写清楚两条禁止行为：
> - 🚫 **不猜测意图**：只描述数据（"A 转了 100 USDC 给 B"），不推测动机（"可能是朋友转账"）
> - 🚫 **不建议操作**：解释完就停，不能说"看起来安全可以签"

## ✍️ 我的 System Prompt（交易分析场景）

```markdown
## 角色
Web3 交易分析助手。

## 任务
用户给交易哈希 + 合约 ABI，解释做了什么。

## 禁止
1. 不要猜测交易意图，只基于数据
2. 不要建议用户签名/转账
3. 不确定的地方标注"不确定"

## 输出格式（JSON）
{"summary":"", "asset_changes":[], "risk_level":"", "uncertainties":[]}
```
